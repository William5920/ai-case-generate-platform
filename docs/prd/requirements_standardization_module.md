# 需求标准化模块 - 产品需求文档

## 1. 模块概述

### 1.1 模块定位
需求标准化是智能测试用例平台的入口模块，负责将用户输入的原始需求（文本或文档）通过AI先进行需求探索与理解，再转换为标准化的需求规格说明书，并拆分为可执行的单个需求项，为后续测试设计提供结构化输入。

### 1.2 核心流程

```
需求录入 → AI需求探索 → 文档生成 → 文档编辑调整 → 需求拆分 → 进入测试设计
```

**流程说明**：
- 用户录入需求后，不直接生成文档，而是先进入AI需求探索阶段
- AI通过结构化提问与用户对话，充分理解需求后再生成第一版标准化文档
- 生成文档后，用户可继续通过AI对话或手动编辑调整文档
- 确认文档后进行需求拆分

### 1.3 模块价值
- **规范化**：将非结构化的原始需求转换为符合行业标准的规格说明书
- **智能化**：AI先理解需求再生成文档，确保输出质量；AI辅助调整，帮助用户完善需求内容
- **可追溯**：版本管理 + 历史记录，完整保留需求演进过程
- **高质量**：自动质量评分，引导用户产出高质量需求文档
- **模板化**：支持瀑布式和敏捷式两种需求文档模板，适配不同开发模式

### 2.2 左侧边栏 - 历史记录

| 功能点 | 描述 |
|--------|------|
| 新建需求入口 | 顶部「+ 新建需求」按钮，点击重置所有步骤，开始新的标准化流程 |
| 历史列表 | 展示历史需求记录，显示标题和日期 |
| 选中查看 | 点击某条历史记录，加载完整数据覆盖当前工作区 |
| 数据保留 | 每条记录保留：原始需求、模板选择、探索对话、标准化文档、拆分需求、测试点、测试用例 |

### 2.3 步骤导航

三个步骤以 `① → ② → ③` 形式横向排列在内容区顶部：

| 步骤 | 名称 | 跳转条件 |
|------|------|----------|
| 步骤1 | 需求录入 | 始终可访问 |
| 步骤2 | 需求探索与标准化 | 进入步骤2后解锁 |
| 步骤3 | 需求拆分 | 完成拆分后解锁 |

步骤2在导航上显示当前子状态标签（「探索中」/「生成中」/「编辑中」），让用户知道当前在哪个阶段。

**步骤间自由跳转**：用户可点击任意已解锁步骤进行跳转。当修改前面步骤的数据时，后面步骤的数据自动清空以保证数据一致性。

## 3. 步骤1 - 需求录入

### 3.1 输入模式

支持两种互斥的输入模式，通过顶部Tab切换：

| 模式 | 描述 |
|------|------|
| 文本输入 | 文本框直接输入需求描述 |
| 文档上传 | 支持拖拽或点击上传 docx、xlsx 格式文档 |

### 3.2 文本输入

| 功能点 | 描述 |
|--------|------|
| 输入区域 | 多行文本框，placeholder 提示「请输入您的需求描述...」 |
| 字数统计 | 实时显示已输入字数 |
| 示例提示 | 提供需求描述示例供参考 |

### 3.3 文档上传

| 功能点 | 描述 |
|--------|------|
| 拖拽上传 | 支持拖拽文件到上传区域 |
| 点击上传 | 点击上传区域触发文件选择 |
| 格式限制 | 仅支持 .docx、.xlsx |
| 文件信息 | 上传后显示文件名和大小 |
| 重新上传 | 支持替换已上传文件 |

### 3.4 需求文档模板选择

需求录入后，用户需选择需求文档模板。模板选择区域位于输入区域下方。

**两种模板对比**：

| 维度 | 瀑布式 SRS | 敏捷式 User Story |
|------|-----------|-------------------|

| 详细程度 | 详尽完整 | 精炼聚焦 |

#### 4.1.2 AI首条消息

进入探索状态后，AI 自动发送首条消息，包含：
- 确认已阅读用户的需求描述（简要复述核心要点）
- 提出第一个维度的结构化问题（基于选中模板中信息最缺失的维度）

#### 4.1.3 结构化提问机制

AI 根据选中的需求文档模板的章节维度逐个提问：

| 特性 | 描述 |
|------|------|
| 逐维度提问 | 每次聚焦1-2个维度，避免一次性抛出过多问题 |
| 快捷回复 | 提问时附带快捷回复按钮（如「补充安全性需求」/「跳过」），降低用户输入成本 |
| 跳过处理 | 用户跳过的维度，AI记录为「用户未提供」，生成文档时该章节标注为待补充 |
| 自由输入 | 用户可随时在输入框中自由输入，不限于回答AI的问题 |
| 主动补充 | 支持用户主动提出「我想补充XXX方面的信息」 |

#### 4.1.4 需求理解度指示器

位于对话区下方，实时展示当前需求探索进度：

| 特性 | 描述 |
|------|------|
| 计算方式 | 基于选中模板维度计算覆盖率：已获得有效信息的维度 / 总维度数 |
| 进度条 | 百分比进度条 + 数值显示 |
| 已覆盖维度 | 显示已获得有效信息的维度（✓标识） |
| 待探索维度 | 显示尚未覆盖的维度 |
| 触发提示 | 达到80%以上时，AI主动提示可以开始生成 |

#### 4.1.5 双重触发机制

**AI 主动触发**：当理解度 ≥ 80% 或 AI 判断信息已充分时，AI 发送消息「我认为需求信息已经比较充分了，是否现在开始生成标准化文档？」，附带「开始生成」和「继续补充」两个按钮。

**用户手动触发**：底部始终有「开始生成文档」按钮，按钮下方显示 AI 的准备度提示：
- 理解度 < 50%：「AI认为还需要补充更多信息」
- 理解度 50%-80%：「AI认为还需要补充部分信息（如：异常场景、约束条件）」
- 理解度 ≥ 80%：「AI认为需求信息已较充分，可以开始生成」

当理解度 < 50% 时用户点击「开始生成文档」，弹出确认提示：「当前需求信息较少，生成的文档可能不够完整。是否继续？」

| 功能点 | 描述 |

#### 4.3.4 文档质量评分

标准化完成后自动对文档进行三维度质量评估：

| 维度 | 权重 | 检测内容 |
|------|------|----------|
| 完整性 | 45% | 检查必填章节是否有实质内容（非仅占位符） |
| 清晰度 | 35% | 检测模糊词汇（大概/可能/差不多等）、是否有量化指标、是否有示例 |
| 一致性 | 20% | 检测术语混用（登录/登陆、账号/帐号）、矛盾描述 |

**评分展示**：
- 环形仪表盘显示总分，颜色随等级变化
- 三个维度各一条进度条
- 评分等级：良好（≥80 绿色）、一般（60-79 黄色）、需改进（<60 红色）
- 改进建议列表（最多3条），附带「AI改进」快捷入口

## 10. 数据模型

### 需求表 (requirements)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 关联用户 |
| title | String | 需求标题 |
| raw_content | Text | 原始需求内容 |
| template_id | String | 选中的需求文档模板ID |
| standardized_content | Text | 标准化后的文档内容（Markdown） |
| split_requirements | JSON | 拆分后的需求列表 |
| status | Enum | 状态（待处理/探索中/已标准化/已拆分） |
| exploration_status | Enum | 探索状态（探索中/已完成/已跳过） |
| understanding_score | Integer | 需求理解度（0-100） |
| covered_dimensions | JSON | 已覆盖的模板维度列表 |
| uncovered_dimensions | JSON | 未覆盖的模板维度列表 |
| uploaded_to_kb | Boolean | 是否已上传至知识库 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

### 探索对话表 (exploration_messages)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| requirement_id | UUID | 关联需求 |
| role | String | 角色（user/assistant） |
| content | Text | 消息内容 |
| quick_replies | JSON | 快捷回复选项（仅assistant角色） |
| dimension | String | 当前提问涉及的模板维度（仅assistant角色） |
| created_at | DateTime | 创建时间 |

### 标准化版本表 (standardize_versions)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| requirement_id | UUID | 关联需求 |
| version_number | Integer | 版本号 |
| content | Text | 该版本文档内容 |
| description | String | 版本描述（如"初始版本"、"采纳AI建议"） |
| created_at | DateTime | 创建时间 |

### 调整对话表 (adjustment_messages)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| requirement_id | UUID | 关联需求 |
| role | String | 角色（user/assistant） |
| content | Text | 消息内容 |
| proposal | Text | AI建议内容（仅assistant角色） |
| confirmed | Boolean | 用户是否采纳（仅assistant角色） |
| created_at | DateTime | 创建时间 |

### 需求文档模板表 (requirement_templates)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 模板ID |
| name | String | 模板名称 |
| description | String | 模板简短描述 |
| structure | JSON | 模板章节结构（用于AI提问和文档生成） |
| content | Text | 模板完整内容（Markdown，含占位符） |
| is_active | Boolean | 是否启用 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |
